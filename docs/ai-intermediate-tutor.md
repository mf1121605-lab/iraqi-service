# المدرس الخصوصي الذكي — الثالث متوسط (مواصفات معمارية)

> حالة الميزة: **متوقفة مؤقتاً** بعد بناء نسخة أولى كاملة (قاعدة بيانات + خادم + واجهة).
> الكود موجود في الفرع الرئيسي (`main`) ولم يُحذف — هذا الملف يوثّق القرارات المعمارية
> النهائية المتفق عليها حتى يمكن استئناف العمل عليها لاحقاً دون إعادة نقاش القرارات نفسها.

## 1. الهدف (Target)

- المنهج المستهدف: منهج الصف **الثالث متوسط** العراقي.
- المواد المدعومة (كل جلسة = مادة واحدة فقط، تُختار صراحةً من واجهة الطالب قبل بدء
  المحادثة، ولا يُترك للنموذج استنتاجها):
  - اللغة العربية (`arabic`)
  - اللغة الإنكليزية (`english`)
  - الرياضيات (`math`)
  - العلوم (`science`)
  - الاجتماعيات (`social_studies`)
  - التربية الإسلامية (`islamic_education`)
- آلية العلامة الصريحة للسياق: يُمرَّر مفتاح المادة (`subject`) من الجلسة المخزَّنة في
  قاعدة البيانات إلى الخادم عند كل رسالة، ويُبنى منه نص تعليمات نظام (system prompt)
  يُلزم النموذج بالبقاء ضمن هذه المادة حصراً ويطلب من الطالب فتح جلسة جديدة لأي مادة
  أخرى بدل الانحراف عنها.

## 2. الشخصية والأسلوب التربوي (Persona)

- النموذج يتصرف كمدرس خصوصي صارم تربوياً (توجيه منهجي واضح، لا مجرد دردشة عامة)،
  يشرح المفاهيم بأسلوب مبسّط يناسب طالباً بعمر 14–15 سنة.
- عند الطلب، يولّد **أسئلة على نمط الامتحان الوزاري** (أسئلة استرشادية للتدريب)
  وملخصات للدروس.
- إخلاء مسؤولية دائم وثابت في تذييل واجهة الدردشة (ليس نصاً يولّده النموذج، بل عنصر
  ثابت في الواجهة): الأسئلة والتمارين استرشادية لأغراض التدريب فقط، وليست أسئلة
  وزارية رسمية معتمدة.
- لاحقاً (خطوة تالية مخطط لها، لم تُنفَّذ بعد): دمج نصوص وزارية حقيقية يزوّدنا بها
  المؤسس كأمثلة (Few-shot) لرفع دقة صياغة الأسئلة التوليدية لتضاهي الصياغة الوزارية
  الفعلية.

## 3. قاعدة البيانات (Database)

جدولان مخصصان (تمت إضافتهما فعلاً عبر ملف الترحيل
`supabase/migrations/20260718180000_ai_tutor.sql`):

### `tutor_chat_sessions`
- `id`, `student_id` (مرتبط بـ `profiles`), `subject` (بقيد `check` على القيم الست
  أعلاه), `title`, `created_at`, `updated_at`.
- جلسة واحدة = مادة واحدة، ثابتة طوال عمر الجلسة.

### `tutor_messages`
- `id`, `session_id`, `student_id` (denormalized لتبسيط استعلامات تحديد سقف
  الاستخدام), `role` (`user` أو `assistant`), `content`, `created_at`.

### الأمان (RLS)
- كلا الجدولين مفعَّل عليهما Row Level Security، وكل السياسات مقيَّدة بـ
  `student_id = auth.uid()` — كل طالب يرى وينشئ جلساته الخاصة فقط.
- **قرار أمني مقصود**: لا توجد سياسة `INSERT` لـ `authenticated` على جدول
  `tutor_messages` إطلاقاً. كل الرسائل (سؤال الطالب ورد النموذج معاً) تُكتب حصراً من
  الخادم عبر مفتاح الخدمة (`service role`)، بعد أن يطبّق الخادم حدود الاستخدام. هذا
  يمنع أي طريقة للطالب لإدراج رسائل مباشرة من المتصفح وتجاوز الحدود المفروضة على
  الاستهلاك المدفوع.

### الترقيم الصفحي (Pagination)
- تحميل أولي لآخر **20 رسالة فقط** لكل جلسة عند فتحها (وليس كامل السجل).
- زر "عرض رسائل أقدم" يجلب الدفعة السابقة (20 رسالة إضافية) عبر مؤشر
  `created_at` بدلاً من الترقيم بالأرقام، لضمان أداء جيد على الشبكات الضعيفة.
- نفس آخر 20 رسالة تُستخدم أيضاً كنافذة السياق (context window) المُرسلة إلى نموذج
  الذكاء الاصطناعي مع كل طلب رد جديد — وليس تاريخ المحادثة كاملاً — لضبط التكلفة
  وزمن الاستجابة.

## 4. ضبط التكلفة (Cost Controls)

- **النموذج**: Claude 3 Haiku (`claude-3-haiku-20240307`) — الخيار الأخف والأوفر
  تكلفة، حسب القرار الصريح للمؤسس، عبر استدعاء مباشر لواجهة Anthropic Messages API
  (بدون مكتبة SDK إضافية).
- **حد طول الرسالة**: 800 حرف كحد أقصى لكل رسالة يرسلها الطالب.
- **تحديد المعدل (Rate Limiting) لكل طالب**:
  - 20 رسالة كحد أقصى في الساعة الواحدة.
  - 60 رسالة كحد أقصى في اليوم الواحد.
  - يُحتسب العدّاد من عدد رسائل `role = 'user'` الفعلية المخزَّنة في `tutor_messages`
    ضمن النافذة الزمنية (بدون جدول عدّاد منفصل).
- **حد زمني على استدعاء النموذج**: مهلة قصوى 30 ثانية لاستدعاء Anthropic API (عبر
  `AbortController`)، مع رسالة خطأ عربية واضحة عند الفشل أو انتهاء المهلة بدل تعليق
  الطلب إلى ما لا نهاية.
- **بوابة تفعيل واضحة**: إن لم يكن متغير البيئة `ANTHROPIC_API_KEY` مضبوطاً على
  الخادم (لأن المؤسس لم يفتح الحساب المدفوع بعد)، يُعاد للطالب مباشرة رسالة "الخدمة
  غير مفعّلة حالياً" بدل خطأ غامض أو تعطّل صامت.

## الملفات ذات الصلة (للمرجعية عند الاستئناف)

- `supabase/migrations/20260718180000_ai_tutor.sql` — الجداول وسياسات RLS.
- `src/lib/tutorSubjects.js` — قائمة المواد الست ودوال ترجمة أسمائها.
- `src/pages/api/tutor/send-message.js` — نقطة الخادم: التحقق من الملكية، تحديد
  المعدل، بناء السياق، استدعاء Anthropic، حفظ الرسائل.
- `src/pages/customer/tutor/index.js` — شاشة اختيار المادة.
- `src/pages/customer/tutor/[subject].js` — قائمة جلسات المادة المختارة + إنشاء
  جلسة جديدة.
- `src/pages/customer/tutor/session/[sessionId].js` — واجهة الدردشة نفسها مع
  الترقيم الصفحي والتذييل التنويهي.

## الخطوتان المتبقيتان لتفعيل الميزة فعلياً (لم تُنفَّذا بعد)

1. تشغيل ملف الترحيل أعلاه في محرر SQL الخاص بـ Supabase (لم يُؤكَّد تنفيذه على
   قاعدة البيانات الفعلية بعد إيقاف الميزة مؤقتاً).
2. فتح حساب Anthropic مدفوع وإضافة مفتاحه كمتغير بيئة `ANTHROPIC_API_KEY` في Vercel.
